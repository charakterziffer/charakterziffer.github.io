Pattern-Library für @cz+

In dieser Datei sind möglichst alle Stile und Vorlagen vorhanden, die in meinem Blog angewendet werden können. Das ist ein interner Link, in diesem Fall ein bereits besuchter. Ein unbesuchter Link sieht so aus. Wie verlinke ich dann einen Seitenanker? Übrigens kann man den Titel des Links auch weglassen.

Falls man keinen Extra-Linktext braucht, verlinkt man einfach so https://direktadresse.de/ (klappt nur bei externen Links). Das geht auch für Mailadressen: address@example.com (mit Spamschutz).

Aus Lesbarkeitsgründen im Markdown-Text kann man auch Referenzlinks verwenden. Sie lagern die Linkadresse einfach an eine beliebige Stelle im Dokument aus, machen im HTML-Quelltext aber keinen Unterschied.

Hervorhebungen (H2-Überschrift)

Kursive Textstellen werden mit einem Stern umschlossen, fette Hervorhebungen bekommen zwei Sterne. Man kann auch Wortbestandteile hervorheben. Falls tatsächlich ein *Sternchen* gemeint ist, entschäft man es mit einem Backslash. Für Tastenkombinationen gibt es den <kbd>-Tag, den man beispielsweise für Strg + + verwenden kann.

Weitere Formatierungen (H3-Überschrift)

Kurze Codeschnippsel zeichnet man mit einem Gravis aus. Praktischerweise werden <html>-Sonderzeichen wie spitze Klammern oder „Kaufmanns-&“ gleich umgewandelt.

Farben

Die Grundfarbe meines Blogs ist Grün, das bei Link-Hover zu Hellgrün wird. Als Akzentfarbe gibt es Lila (z.B. für besuchte Links), sowie Rot für Hervorhebungen.

Grundtexte und kleine Titel bekommen ein Antrazit, das für Kursiv schwarz und für größere Titel etwas heller wird. Außerdem verwende ich hellere Grautöne fürs Datum und für Trennlinien.

Für Hinterlegungen stehen Blassgrün und Blasslila zur Verfügung. Dazu Grautöne für Bilduntertitel und etwas heller und halbtransparent für Code, Zitate und Aside.

Kästen, Zitatblöcke, Codeblöcke

Manchmal will ich einen Absatz farblich hervorheben. Dazu gibt es die CSS-Klasse hinterlegt.

Will man einen Absatz besonders hervorheben, dann verpasst man ihm die Klasse .hinterlegt und er steht in einem grünen Kasten.

Man kann auch mehrere Absätze hinterlegen. Dazu packt man sie gemeinsam in ein div. Dann muss man natürlich reines HTML verwenden, denn mit Markdown allein kommt man da nicht weiter.

Zitatblöcke beginnen einfach mit einer spitzen Klammer-zu (>):

So ein Zitatblock wird dann mit <blockquote> umschlossen. Der Text darinnen steht in einem <p>. Das wird zum Beispiel für Tweets benutzt.

Bei Tweetsammlungen stehen meist mehrere Zitatblöcke hintereinander:

⏣ »Manchmal muss du deinen gewohnten Bereich verlassen«, sprach der #Fuchs zur Ballerina. »Nur so entwickelst du dich weiter.« (#)

☘ »Wer nicht von Zeit zu Zeit riskiert, unglücklich zu werden«, sprach der #Fuchs zur Ballerina, »wird auch niemals richtig glücklich sein.« (#)

⌬ »Klar kannst du gegen deine Prinzipien handeln«, sprach der #Fuchs zur Ballerina. »Du solltest aber nen verdammt guten Grund dazu haben!« (#)

Die Überschrift bekommt dabei einen klitzekleinen Abstand oben, damit die Ränder einigermaßen passen. Die graue Markierung von Codeschnippseln ist hier allerdings nur sehr subtil …

Wenn man ein längeres Codebeispiel hat, dann verwendet man am besten einen Codeblock (einfach mit Tab oder mindestens vier Leerzeichen einrücken):

@font-face {
   font-family: 'Schriftname';
   src: url('schrift.eot'); /* IE9 compatibility mode */
   src: url('schrift.eot?#iefix') format('embedded-opentype'),
      url('schrift.woff') format('woff'),
      url('schrift.ttf') format('truetype');
}

Nun folgt eine waagrechte Trennlinie, erzeugt mit drei Bindestrichen.

Alternativ erreicht man eine Linie auch mit mindestens drei Sternchen *, wahlweise mit Leerzeichen dazwischen.
